자바스크립트(Javascript)는 웹 브라우저에서 동작하는 스크립트 언어다. 뭔가 부족한 듯한 소개지만 사실상 모든 것을 대변하는 말이다. 물론 현재는 웹 브라우저 뿐만 아니라 엔진만 떼어내면 뭐든 할 수 있는 플랫폼으로 인식될 정도로 많이 쓰이는 언어이긴 하다.

이 글에서는 Javascript 뿐만 아니라 ECMAScript를 함께 다룬다. 개념 상 이 둘은 다르게 취급하는 것이 맞겠지만, 현재로써 이 둘을 별도로 생각하는 것도 좀 이상한 것 같기도 하다. 버전은 ES6를 기준으로 한다.

노트

여기에 붙는 글들은 정해진 주제 없이 필요할 때 마다 붙는 메모 형식으로 작성됩니다. 후에 글이 모이면 별도의 글로 분리될 수도 있습니다.

모듈(Module)

일반적으로 모듈은 아래와 같은 형식으로 쓰인다. 

// someModule.js
export const name = 'some value';

// someCode.js
import { name } from 'someModule.js';

이름 자체를 주지 않고 default로 export 한 경우라면 이름을 마음껏 지을 수 있다. 

// someModule.js
export default 'default value';

// someCode.js
import anyName from 'someModule.js';

이 두 가지 방식을 섞어서 쓸 수도 있다. 

// someModule.js
export const name = 'some value';
export default 'default value';

// someCode.js
import anyName, { name } from 'someModule.js';

위에서 알 수도 있겠지만, 당연하게도 모듈은 여러 심볼을 가질 수 있다. 

// someModule.js
export const value1 = 'some value 1';
export const value2 = 'some value 2';

// someCode.js
import * as someValues from 'someModule.js';
console.log(someValues.value1);
console.log(someValues.value2);

스프레드(Spreads) 연산자

스프레드 연산자는 ... 으로 표기되며 이름처럼 펼치거나 확산한다라는 의미로 동작한다. 

let arr = [1, 2, 3];
let combi = [...arr, 4, 5, 6];

위 코드에서 combi 는 [1, 2, 3, 4, 5, 6] 이 된다.

이 펼쳐지는 것이 굉장히 특이한데, 배열로 감싸져 있으면 위 처럼 배열의 원소 형태로 펼쳐지지만, 배열이 아니라 그대로 사용하면 코드에 그대로 펼쳐지는 듯이 작동한다. 

function sixArgs(a, b, c, d, e, f) {
    // ...
}

sixArgs(...arr);

이렇게 하면 각 매개변수에 해당 배열의 값들이 알아서 펼쳐져서 들어간다. 즉 a 에는 1 이, b 에는 2 가, 그리고 최종적으로 f 에는 6 이 들어간다.

그래서 다양할 활용이 가능하다. 

[a, b, ...others] = [1, 2, 3, 4, 5, 6]

위 코드가 실행되면 a 에는 1 이, b 에는 2 가, 그리고 others 에는 [3, 4, 5, 6] 이 들어간다.

함수 매개변수로 사용할 수도 있다.1 

function someFunc(value, ...rest) {
    console.log(value);
    console.log(rest);
}

위 함수를 someFunc(1, 2, 3, 4, 5) 로 호출한 경우 value 에는 1 이, rest에는 [2, 3, 4, 5] 가 들어간다.

🀱 각주 (Footnotes)

1

이 경우 스프레드(Spreads)가 아니라 레스트 매개변수(Rest Parameter) 라 불린다.
