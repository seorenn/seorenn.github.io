이 글은 Doom Emacs의 Python 프로그래밍 환경을 위한 모듈의 소개와 설정 및 사용법 등을 정리한다.

참고로 macOS 기준이며 다른 OS의 경우 차이가 있을 수 있다.

Python 준비

당연하겠지만 Python이 미리 설치되어 있어야 한다. macOS의 경우 시스템에 기본 설치가 되어있긴 한데 문제는 2.7이라는 이제는 무덤에 들어간 버전이라 최신의 3.x 버전을 설치하는 편이 좋다. Doom Emacs에서는 기본적으로 python3 라는 실행 파일을 먼저 찾기 때문이기도 하다.

설치 자체는 pyenv로 설치하는 것을 추천한다. pyenv로 파이썬 버전 관리하기 글을 참고해서 설치해보자.

Homebrew 버전은 의존 패키지로만 활용하는 것이 좋을 것 같다. 일부 유틸리티형 패키지의 경로가 제대로 안 잡히는 등 상당히 까다로운 문제를 겪었다.

init.el

Doom Emacs의 가장 기본이 되는 모듈 설정이다. 개인적으로 문법 체크를 위한 syntax 모듈은 있는 편이 좋다고 생각된다.

아래는 개인적으로 사용하는 모듈과 플래그 설정 중 이 글과 관련된 부분만 뽑아놓은 것이다.

:checkers
syntax

:lang
(python
 +lsp
 +pyenv
 +poetry)

보다시피 LSP(Language Server Protocol)와 pyenv, Poetry 지원을 추가로 설정하고 있다.

config.el

위 모듈 설정에서 LSP를 사용할거라고 설정했으니 아래 코드가 config.el 의 어딘가에 들어가 있는 편이 좋다.

(after! lsp-python-ms
  (set-lsp-priority! 'mspyls 1))

이제 LSP 서버의 설치가 필요하다.

LSP Server

Doom Emacs에서는 Microsoft Python Language Server를 추천하는 것 같다. 그 외의 서버는 선택지가 보이지 않았다.

설치는 의외로 단순하다. SPC : 혹은 M-x 를 눌러서 lsp-install-server 함수를 실행시키면 선택지가 뜬다. 여기서 mspyls 를 선택하면 알아서 설치가 된다.

물론 dotnet sdk를 설치하고 수동으로 빌드해서 환경 설정 잡고… 하는 방법이 있긴 한데 상당히 귀찮을거라 생각된다.

필요한 Python 패키지들

이제 아래 커맨드로 각종 Python 패키지를 설치하자.

pip install flake8 isort nose pytest pyflakes

이 목록은 위에서 쓰겠다고 한 플래그 설정과 관련이 있다. 필요하다면 외부 링크에 링크한 공식 매뉴얼을 참고하자.

REPL

Doom Emacs의 Python 모듈은 생각보다 기능을 많이 제공하지 않는다. 아무래도 많은 부분을 수작업으로 관리해야 할 것 같다.

그나마 REPL을 띄우는 기능이 제공되긴 한다. 아래의 세 가지 방법으로 REPL 팝업을 띄울 수 있다.

  • M-x +python/open-repl
  • M-x +python/open-ipython-repl
  • M-x +python/open-jupyter-repl

다른 기능은 알아내거나 구현하는대로 이 글에 업데이트할 예정이다.
