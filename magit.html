It's Magit! A Git Porcelain inside Emacs

에 그러니까… Emacs용 Git 프론트엔드/클라이언트 확장입니다. 개인적으로는 Emacs의 킬러 소프트웨어 중 하나로 꼽을 정도로 막강합니다.

참고로 이 글은 순수 magit이 아닌 Doom Emacs나 Spacemacs용으로 커스터마이징 된 단축키를 위주로 정리합니다.

magit-status

현재 상태를 표시하는 UI를 띄웁니다. 개인적으로 magit 을 시작하는 가장 좋은 방법으로 사용합니다.

함수이므로 당연히 M-x 명령으로 실행시킬 수도 있습니다. Doom Emacs 에서는 SPC g g, Spacemacs 에서는 SPC g s 단축키가 기본 바인딩 되어 있습니다.

아래의 단축키들은 모두 이 화면에서 시작됩니다.

내용을 갱신(refresh)하기 위해서는 g r 을 입력하면 됩니다.

Stage

수정 내역에서 s 를 누르면 스테이징이 됩니다. 각 수정내역에서 TAB 을 누르면 간단한 diff를 볼 수 있는데, 여기서 선택영역을 지정하고 s 를 누르면 선택 영역만 따로 스테이징이 됩니다.

타이틀 부분을 선택하고 s 를 누르면 해당 분류의 파일들이 모두 스테이지 됩니다.

Commit

c c 를 누르면 커밋 모드로 돌입합니다. 스테이징 된 파일들의 간략한 변경점을 볼 수 있으며 커밋 메시지를 작성할 수 있습니다.

커밋메세지를 작성한 이후에는 다른 방법으로 커밋을 진행하는데, Doom Emacs 의 경우 :wq 로 vim 을 종료하듯이 하면 커밋이 진행됩니다. Spacemacs 의 경우 ,c 와 같은 별도의 단축키가 배정되어 있으며 이 부분은 아래쪽에 주석으로 도움말이 적혀있으니 참고합시다.

Log

l l 을 이용하면 로그를 볼 수 있습니다.

각 커밋에서 RET 를 누르면 해당 커밋의 변경 내역을 볼 수도 있습니다.

Push

p p 를 이용하면 원격에 현재 브랜치 이름으로 푸시가 됩니다.

Pull

F u 를 이용하면 origin 에서 현재 브랜치로 풀이 됩니다.

Branch

  • b c 로 브랜치를 생성합니다.
  • b b 로 브랜치를 전환합니다.
  • b x 로 브랜치를 삭제합니다.

기타

  • magit-status 에서 $ 키를 누르면 실제로 사용되는 커맨드 및 응답 등의 로그를 볼 수 있습니다.
  • Doom Emacs에서 SPC g s1 로 현재 파일의 변화내용을 스테이지할 수 있습니다.
