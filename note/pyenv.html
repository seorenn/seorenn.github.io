pyenv는 Python 버전을 관리하고 virtualenv 환경을 통합해서 관리해주는 기능을 제공하는 도구다. 시스템에 원하는 Python 버전을 설치하고 특히 다중 버전을 동시에 설치하고 실행시키기 좋은 환경을 제공해 준다.

설치

macOS 에서는 Homebrew 로 설치하는 것이 편하다. 터미널에서 아래 커맨드로 설치할 수 있다. 

brew install pyenv

기본 패키지 설치는 쉽게 완료되는데, 최근 버전에서는 버전 리스트 관리 방식이 바뀌었는지 후속 설치를 하는 것이 좋을 것 같다. 역시 터미널에서 아래 커맨드를 입력한다. 

git clone https://github.com/pyenv/pyenv.git ~/.pyenv

이미 기존이 ~/.pyenv 디렉토리가 있을 경우 설치가 안 될 수도 있으니 미리 삭제하고 진행하자.

이제 셸 환경변수를 몇 가지 설정해야 한다. 공식 사이트에서는 아래 커맨드를 이용하라고 한다. 

echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.bash_profile
echo 'export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.bash_profile
echo -e 'if command -v pyenv 1>/dev/null 2>&1; then\n  eval "$(pyenv init -)"\nfi' >> ~/.bash_profile

물론 사용하는 셸에 따라서 대상 파일 이름이 달라진다는 점에 주의하자. 개인적으로는 zsh을 쓰기 때문에 ~/.zshrc 파일에 아래와 같은 내용을 넣었다. 

export PYENV_ROOT="$HOME/.pyenv"
export PATH="$PYENV_ROOT/bin:$PATH"
eval "$(pyenv init -)"
eval "$(pyenv virtualenv-init -)"

가장 마지막 라인은 공식 사이트에서는 소개하고 있지는 않은데 예전부터 쓰고 있는 커맨드라서 넣어 놨다. 혹시나 virtualenv 관련 기능이 동작하지 않는다면 마지막 라인도 추가해서 테스트 해보자.

이후 셸을 새로 기동하자. 혹은 터미널을 껐다 켜도 되고 source 등의 커맨드로 다시 스크립트를 로딩 해도 관계는 없다.

버전 목록

아래 커맨드를 이용해 설치할 수 있는 버전 목록을 얻을 수 있다. 

pyenv install --list

상당히 긴 리스트가 나오는 것에 유의하자. 아마도 대부분의 경우 상단에 별 다른 머릿 글자 없이 버전 숫자만 표기된 버전이 필요할 것이다.

파이썬 설치하기

원하는 버전을 설치하려면, 예를 들어 3.8.1 버전을 설치하려면 아래와 같은 커맨드를 사용하자. 

pyenv install 3.8.1

이렇게 하면 3.8.1 버전이 설치된다.

버전 관리자인 만큼 당연하게도 여러 버전을 설치할 수 있다. 

pyenv install 2.7.13

2.7 버전은 이제는 쓰지 말아야 하겠지만 어쨌든 이것은 예제다.

설치된 버전은 versions 명령으로 확인할 수 있다. 

pyenv versions

기본 실행 버전 설정

일단 pyenv로 파이썬을 설치 했더라도 바로 사용할 수는 없다. pyenv root 디렉터리에 설치가 되었기 때문이다. 따라서 시스템 기본 파이썬으로 만들어 줄 필요가 있다.

아래 커맨드는 파이썬 3.8.1 버전을 기본 버전으로 설정한다. 

pyenv global 3.8.1

혹시 두 종류 이상의 버전이 설치되어 있다면, 예를 들어 python 2.7.13을 보조로 같이 사용하고 싶다면 아래와 같이 설정할 수도 있다. 

pyenv global 3.8.1 2.7.13

이렇게 할 경우 python 커맨드로 실행되는 버전은 3.8.1이고, python2 명령으로 실행되는 버전은 2.7.13이 된다.

virtualenv 지원

파이썬을 새로 설치했다면 virtualenv 가 없을 테니 별도로 설치하자. 

pip install virtualenv

virtualenv 환경을 만들기 위해서는 아래와 같은 식의 커맨드를 입력할 수 있다. 

pyenv virtualenv 3.8.1 my_project_env

이 명령은 파이썬 3.8.1 버전을 기준으로 myproject 라는 virtualenv 환경을 생성하는 명령이다.

이제 이 환경을 셸로 가져오기 위해서 아래 커맨드를 이용할 수 있다. 

pyenv shell my_project_env

이렇게 하면 해당 환경으로 셸이 뜨며 여기서 pip 등으로 필요한 패키지를 설치하고 실행시켜 볼 수 있다.

설치 목록 업데이트

가끔 새 버전 파이썬이 나왔는데 pyenv의 설치 목록에는 아직 구 버전만 보인다면 리스트가 업데이트 되지 않았을 확률이 크다.

아래 커맨드를 이용하면 리스트를 업데이트 할 수 있다. 

cd $(pyenv root) && git pull

이렇게 한 뒤에 pyenv install --list 명령으로 최신 버전이 보이는지 확인해보자.

참고로 앞서 설치 과정에서 git clone을 해두지 않았다면 이 과정이 안 될 수도 있다.
