kubectl

kubectl 은 Kubernetes 클러스터 컨트롤 도구입니다. 이 문서에서는 이 kubectl 의 사용법 예를 몇 가지 정리합니다.

사용법 예제

포드 리스트

kubectl get pods

포드 정보 보기

kubectl describe pod POD_NAME

YAML 파일 적용

kubectl apply -f foo/bar.yaml

포드 삭제하기

보통은 리스케일로 0대만 동작하게 설정하는 식으로 포드를 종료시킵니다. 

kubectl scale --replicas=0 deployment/POD_NAME

YAML 파일이 있으면 포드 이름을 찾을 필요 없이 좀 더 쉽게 됩니다. 

kubectl scale --replicas=0 -f foo/bar.yaml

아예 포드를 삭제시킬 수도 있습니다. Deployment가 아닌 포드는 이 방법으로 삭제하는 것이 편하기도 합니다. 

kubectl delete pod POD_NAME

포드 재시작하기

특별히 커맨드가 있는건 아니고 아래처럼 할 수 있습니다. 

kubectl edit deployment/POD_NAME

이렇게 입력하면 에디터가 뜨는데 그냥 저장하고 종료하면 재시작 된다고 합니다. (즉 시험 안 해봄😛)

로그 보기

kubectl logs -f POD_NAME

보통 사용하는 tail -f 와 비슷한 커맨드입니다.

특정 앱의 모든 로그를 보려면 아래처럼 할 수 있습니다. 

kubectl logs -f -l app=APP_NAME

기본적으로 5개의 포드 까지만 볼 수 있게 되어있는데 에러메시지에 옵션(--max-log-requests) 정보가 나오니까 이것도 참고해 둡시다.

포트 포워딩

kubectl port-forward POD_NAME PORT_NUM

PORT_NUM 으로 표기한 부분은 포트 번호인데 HOST_PORT:CONTAINER_PORT 와 같은 식으로 다르게 지정하는 것도 가능합니다.

포드가 아닌 서비스도 포트 포워딩이 됩니다. 

kubectl port-forward svc/SERVICE_NAME PORT_NUM

컨테이너의 커맨드 실행시키기

kubectl exec -it POD_NAME -- COMMAND

-it 옵션을 넣었기 때문에 커맨드에 셸을 넣으면 당연히 셸도 뜹니다.

포드가 하나 이상의 멀티 컨테이너일 경우 --container1 옵션으로 골라서 접속할 수 있습니다. 

kubectl exec -it POD_NAME --container CONTAINER_NAME -- COMMAND

심화 예제

레코드와 롤백

--record 커맨드로 현재 기록을 남겨둘 수 있습니다. 

kubectl apply -f somedeploy.yaml --record

그리고 롤백도 쉽게 됩니다. 

kubectl rollout history deploymenys some-deployment

아직 시험은 안 해봤습니다.😏
