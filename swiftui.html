컴포넌트

HStack and VStack

HStack {
    VStack(spacing: 4) {
        Text("foo")
        Text("bar")
    }

    VStack(alignment: .leading) {
        ...
    }
}

HStack의 경우 베이스 라인의 기준을 정할 수도 있습니다. 

HStack(alignment: .lastTextBaseline) {
    Text("caption").font(.caption)
    Image("20x20")
    Text("Title")
}

위 코드는 마지막 텍스트의 하단을 기준으로 베이스 라인을 맞춰줍니다.

참고로 HStack 의 경우 오른쪽에서 왼쪽으로 읽는 문자 표시 시에는 순서를 알아서 반대로 표시해 주는 듯 합니다.

ZStack

ZStack은 UI를 바닥부터 쌓아가는 형태로 만들 수 있게 해 줍니다. 대충 쌓기를 하고 있는데 위에서 쳐다본다고 생각합시다. 

ZStack {
    ForEach(...) {
        ...
    }
}

Spacer

플렉시블한 공백입니다. 즉 컴포넌트와 컴포넌트 사이에 가능한 최대한의 공간을 넣는데 사용합니다. 

Spacer()

List

가장 기본적인 정적 리스트는 아래와 같은 방식입니다. 

List {
    SomeRowView()
    SomeRowView()
}

데이터 모델을 사용한 동적 리스트는 아래와 같은 식이 되겠네요. 

List(someItemList, id: \.id) { item in
    SomeRowView(data: item)
}

해당 데이터가 각 아이템 타입이 Identifiable 프로토콜을 따른다면 id 필드를 생략할 수도 있습니다. 관련 내용은 Swift 5.1 글에서 참고해 봅시다.

NavigationView

기존의 Navigation Controller와 비슷한 역할을 하는 컴포넌트입니다. 

NavigationView {
    List(someItemList) {
        NavigationLink(destination: SomeDetailView()) {
            SomeRowView()
        }
    }
    .navigationBarTitle(Text("Foo Bar"))
}

타이틀은 여러 옵션이 있을 수 있습니다. 

.navigationBarTitle(Text("foo bar"), displayMode: .inline)

레이아웃 속성들

frame

Image("image_20x20").frame(width: 30, height: 30)

padding

기본 패딩 주기로 쓸 수 있습니다. 

Text("foo bar").padding()

물론 어떤 방향에 패딩을 줄지 고를 수 있습니다. 

Text("foo bar").padding([.leading, .trailing])

패딩 수치도 당연히 고를 수 있습니다. 

Text("foo bar").padding(10)

특정 한 방향에 음수로도 줄 수 있는 트릭도 있습니다. 

Text("foo bar").padding(.bottom, -130)

lineLimit

Text("foo bar").font(.caption).lineLimit(1)

참고로 Stack에도 쓸 수 있습니다. 

HStack {
    ...
}
.lineLimit(1)

layoutPriority

HStack {
    Text("foo")
    Text("bar").layoutPriority(1)
}
.lineLimit(1)

우선순위가 낮으면 텍스트가 넘칠 경우 ... 으로 줄여질 수도 있겠지요?

alignmentGuide

베이스라인을 수동으로 맞춰줍니다. 

Image("some_image").alignmentGuide(.lastTextBaseline) {
    d in d[.bottom] * 0.927
}

aspectRatio

SomeView {
    ...
}
.aspectRatio(1, contentMode: .fit)

offset

레이아웃 위치에서 임의로 벗어나게 만들 수 있습니다. 

Circle().offset(y: -130)

스타일 속성들

background

Text("foo bar").background(Color.green)

font

Text("foo").font(.title)
Text("bar").font(.caption)

fill

쉐이프에 색칠을 합니다. 

Circle().fill(Color.red)

stroke

strokeBorder

Circle().strokeBorder(Color.red, lineWidth: 50)

clipShape

image.clipShape(Circle())

overlay

image.overlay(Circle().stroke(Color.white, lineWidth: 4))

shadow

image.shadow(radius: 10)

Text("foo").tapAction {
    ...
}

Gradient

LinearGradient

LinearGradient(gradient: ..., startPoint: ..., endPoint: ...)

AngularGradient

AngularGradient(gradient: Gradient(...),
                center: .center,
                angle: .degrees(-90))

center 점을 중앙으로 삼고 둥글게 그라디언트를 그립니다. 어디에다 그리느냐에 따라 최종 형상이 달라집니다. 

Circle().fill(someGradient)

위 코드로 눈치챌 수 있지만 그라디언트는 색상 대신 사용할 수 있습니다.

Path

var p = Path()
p.addArc(center: ..., radius: ..., startAngle: ..., endAngle: ..., clockwise: false)
p.addLine(to: ...)
...
p.closeSubpath()

아래와 같은 식이 좀 더 SwiftUI 답긴 합니다. 

Path { path in
    path.move(to: ...)
    path.addLine(to: ...)
    path.addQuadCurve(to: ..., control: ...)
    ...
}
.fill(Color.black)
