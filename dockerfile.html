Dockerfile

Seorenn Static Pages

Dockerfile 은 Docker 용 이미지를 빌드하기 위한 특수한 DSL 스크립트 언어로 작성하는 파일입니다.

평범하게 이미지 만들기

FROM debian:buster

RUN apt-get update && \
    apt-get install -y build-essential python-dev python-pip

ADD . /app

RUN cd /app && \
    make && \
    make install && \
    pip install -r requirements.txt

ENV APP_NAME "sample"
ENV APP_VERSION "1.0"

ENTRYPOINT ["/usr/local/bin/app"]

이 이미지는 debian:buster 라는 리눅스 컨테이너에서 몇 가지 빌드에 필요한 도구들과 파이썬 도구들을 설치한 후 실제 앱을 빌드하고 실행시키는 내용을 포함하고 있습니다.

RUN 커맨드를 여러 줄에 나누어서 적어도 관계 없지만, 왜때문인지(?) 길이에 한계가 있기 때문에 이런 식으로 한 줄에 몰아서 적는 형태를 많이들 사용합니다.

ADD 는 현재 빌드가 진행되고 있는 디렉토리의 내용물을 이미지에 몽땅 복사합니다. 어찌보면 가장 중요한 명령어지요.

ENTRYPOINT 는 실제로 이미지가 기동되면 실행되는 쉘 커맨드를 기입하는 곳입니다. 이것 대신 CMD 로 기입할 수도 있는데, 이 경우 사용자 커맨드로 컨테이너를 띄울 경우 CMD 로 지정된 명령어는 실행되지 않습니다. 반대로 ENTRYPOINT 의 경우 사용자 커맨드 여부와 관계 없이 무조건 실행됩니다.

유저

특정 유저로 굴리고(?) 싶다면 USER 커맨드를 쓸 수 있습니다. 

RUN groupadd -r someuser
RUN useradd -m -r -g someuser someuser
USER someuser

컨테이너가 root 유저 기반으로 돌아가니 꺼릴 게 없습니다.
