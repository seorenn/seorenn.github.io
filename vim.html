소개

vi 에디터는 아주 오래전 UNIX용 편집기로 탄생한 텍스트 에디터입니다. 그리고 Vim(Vi Improved)은 vi 에디터의 개선판으로 일단은 vi 와 Vim은 같지 않다라고 보는게 정석입니다.

이 글은 Vim 설명서나 튜토리얼이 아니라 그냥 유명한 몇 가지 관련 항목만 정리합니다.

Vim에 관한 몇 가지 팁

커서키

일반적인 커서 이동키를 사용할 수도 있겠지만, vi나 Vim은 h, j, k, l 의 특수한 커서 이동키를 주로 사용합니다. 이 키는 아래와 같은 커서 이동키 배치로 생각할 수 있습니다. 

  k
h   l
  j

한번 익숙해지면 헤어나오기 힘든 악명 높은(?) 키입니다. 심지어 여기저기에서 차용하고 있는 커서 대안 이동키이기도 합니다. 특히 j 키와 k 키는 페이스북이나 트위터 등 다양한 서비스에서 비슷한 용도로 사용할 수 있기도 합니다.

종료 방법

뜬금없지만 vi 혹은 Vim 과 관련된 글에서 빠지지 않는 주제가 바로 이 종료 방법입니다. 실수로 vi를 열었을 때 종료하지 못 해서 우왕자왕하는 사태가 발생했다는 이야기는 유명하지요.

vi 혹은 Vim의 종료 방법은 보통 아래처럼 설명합니다. 

:q

이를 풀어서 해설하자면:

  • ESC 키를 누른 후
  • : 를 누르고
  • q 를 누르고
  • 엔터를 누르면

종료가 됩니다.

참고로 ESC키가 없는 특정 맥북류에서는 ESC 대신 ^[ 를 눌러도 됩니다. 이 키는 Ctrl+[ 와 동일한 의미입니다.

실수로 운 좋게(?) 편집을 했다면 강제 종료의 의미로 느낌표를 붙이면 저장 없이 그대로 종료가 됩니다. 

:q!

즉 풀어서 설명하자면:

  • ESC 를 누르고
  • : q ! 를 차례대로 누른 후
  • 엔터를 누르면

강제종료가 됩니다.

Undo/Redo

  • u Undo (소문자 u 주의)
  • U Undo current line (현재 라인의 Undo, 대문자 U 주의)
  • ^R Redo (Ctrl+R)

점프(Jump)

  • f [character] 현재 라인에서 해당 문자로 점프
  • F [character] 현재 라인에서 해당 문자로 역방향 점프

인코딩

맥 등 윈도가 아닌 OS에서 만약 텍스트 파일을 열었는데 한글이 깨져 보인다면 아래 커맨드를 시험해봅시다. 

:e ++enc=euc-kr

잘 보이나요? 만약 안 된다면 .vimrc 파일에서 fencs 설정이 어떤지 확인해 봅시다. 

set fencs=ucs-bom,utf-8,euc-kr,cp949

이런 내용이 있다면 아마도 되어야 정상일 것입니다. 없다면 추가해 줍시다. 참고로 euc-kr 과 cp949 는 사실상 같은 거라 맞는 쪽으로 하나만 써도 됩니다.

파일을 다시 utf-8 으로 저장하고 싶다면 비슷하게 아래 커맨드를 사용해 볼 수 있습니다. 

:w ++enc=euc-kr
