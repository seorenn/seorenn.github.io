Homebrew

Homebrew는 macOS용 패키지 관리자입니다. 기본 유틸리티가 아닌 다양한 소프트웨어를 내려받아 자동으로 설치할 수 있게 도와줍니다. 또한 root 사용자가 아니어도 사용할 수도 있도록 구조를 만들어 준다는 점도 특징입니다.

설치

쉘에서 아래 커맨드를 복사해서 붙여 넣습니다. 

/usr/bin/ruby -e \
"$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

curl 유틸리티가 없으면 안 될 수도 있습니다. 기타 문제가 있다면 🌏 공식 홈페이지를 확인해 봅시다.

기본 사용법

아래 커맨드로 패키지를 검색할 수 있습니다. 

brew search KEYWORD

각 패키지의 자세한 정보를 보려면 info 커맨드를 사용합니다. 

brew info PACKAGE_NAME

패키지를 설치하려면 install 명령을 사용합니다. 

brew install PACKAGE_NAME

과거에는 update 를 하지 않으면 최신 버전 정보가 없어서 설치에 문제가 있곤 했는데 요즘은 별로 신경 쓸 필요가 없는 것 같습니다.

설치된 패키지의 최신 버전이 존재하는지 확인하려면 update 후 outdated 를 살펴봅시다. 

brew update
brew outdated

패키지를 최신 버전으로 업그레이드하려면 upgrade 명령을 사용합니다. 

brew upgrade PACKAGE_NAME

모든 패키지를 업그레이드하려면 패키지 이름을 생략하면 됩니다. 

brew upgrade

설치된 패키지에 따라 굉장히 오랜 시간이 걸릴 수도 있으니 위 커맨드 실행 전에는 마음의 준비를 합시다.

패키지를 업그레이드해도 구버전은 그대로 남아 있습니다. 이를 청소하기 위해 cleanup 커맨드를 사용할 수 있습니다. 

brew cleanup

디스크 공간이 부족할 때 등의 경우에 청소해 주면 됩니다.

패키지를 삭제하려면 uninstall 커맨드를 이용할 수 있습니다. 

brew uninstall PACKAGE_NAME

만약 문제가 있는 것 같다면 의사양반(?)을 호출해 봅시다. 

brew doctor

문제가 보이면 잘 해결해 봅시다.

구 버전 패키지 사용하기

만약 upgrade 등으로 최신버전이 설치되었다면 해당 패키지의 구 버전도 여전히 남아있습니다. 이 경우 구 버전을 사용하고 싶자면 switch 커맨드를 사용할 수 있습니다. 

brew switch PACKAGE_NAME VERSION

이 경우 정확한 버전 번호를 입력해야 합니다.

다중 버전을 제공하는 패키지의 경우

다중 버전의 패키지가 필요한 경우 우선 지원되는지 부터 확인해봐야 합니다. 예를 들어 node.js 의 경우 LTS 버전이 다양하게 제공되고 있습니다. search 커맨드로 어떤 버전이 있는지 확인해 볼 수 있습니다. 

$ brew search node
==> Formulae
node  node@12  node@10  node@8
...

위 내용은 연출한 것으로 실제는 저것보다 훨씬 많은 내용이 검색될 수 있습니다. 어쨌든 위 예에서는 node의 8, 10, 12, 그리고 최신 버전 등이 제공되고 있음을 확인할 수 있습니다.

현재는 최신 버전 node 패키지가 설치된 상태라 가정하고 여기서 추가로 10 버전을 설치해 보겠습니다. 

brew install node@10

그냥 설치는 쉽게 됩니다.

만약 위의 패키지 하나만 설치했다면 실행에는 딱히 문제는 없을 것입니다. 하지만 최신 버전이 설치된 상태로 구버전을 설치했다면 최신 버전이 여전히 실행 대상이라 구버전을 실행시킬 수 있는 방법은 없을 것입니다. 

$ node --version
v13.x.x

구버전이든 최신버전이든 같은 파일명을 사용하기 때문에 공존시키기는 힘들 것입니다. 따라서 만약 10 버전을 쓰고 싶다면 최신 버전을 unlink 시켜야 합니다. 

brew unlink node

그리고 사용하려는 버전을 링크합니다. 

brew link node@10 --overwrite

--overwrite 옵션을 사용하는 이유는 unlink 후에도 남는 파일들이 새로 link 하려는 파일들과 중복되기 때문입니다.

위의 커맨드를 실행시키면 아래 메시지를 볼 수도 있습니다. 

Warning: node@10 is keg-only and must be linked with --force

If you need to have this software first in your PATH instead consider running:
  echo 'export PATH="/usr/local/opt/node@10/bin:$PATH"' >> ~/.zshrc

위의 메시지는 새로 링크한 node@10의 바이너리가 표준 위치가 아닌 다른 곳에 있기 때문에 PATH 환경 변수에 이 위치를 추가해줘야 실행시킬 수 있다는 말입니다. 따라서 아래에 echo로 시작되는 라인을 복사해서 그대로 실행시키면 됩니다.

메시지의 내용은 어떤 쉘(shell)을 쓰는지 등에 딸라 달라질 수 있습니다. 어쨌든 메시지 내용에 맞게 대응하면 됩니다.

이제 버전이 제대로 지정되었는지 확인해봅시다. 

$ node --version
v10.x.x

원하는 버전이 나오면 성공입니다.

최신버전으로 돌리는 경우도 같이 구버전을 unlink한 후 최신버전을 link하면 됩니다만 이 경우에는 PATH에 뭔가를 등록하는 절차는 필요 없겠지요.

Cask

cask는 macOS용 응용 소프트웨어나 다양한 리소스를 설치할 수 있게 도와주는 확장 도구입니다.

사용법은 비슷합니다. 아래 커맨드로 macOS용 유명한 터미널인 iTerm 2 의 정보를 찾아봅시다. 

brew cask info iterm2

필요한 소프트웨어라면 설치해봅시다. 

brew cask install iterm2

설치된 이후에는 소프트웨어에 따라 후속 절차가 필요할 수 있습니다. 메시지에 따라 적당히 따라주면 됩니다.

불행히도 search 커맨드는 사용할 수 없지만, outdated나 upgrade, uninstall 등등의 커맨드는 비슷하게 사용할 수 있습니다.
