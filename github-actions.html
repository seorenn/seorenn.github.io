Github Actions

깃헙 액션은 🌏 Github 에서 제공하는 CI1로, 쉽게 말해서 해당 저장소에 뭔가의 이벤트가 발생하면 어떤 동작을 하도록 만들 수 있는 기능입니다. 예를 들어서 push가 일어나면 이를 빌드해서 배포하는 기능을 자동화 할 수 있지요.

사용법

프로젝트 저장소에서 아래의 디렉토리가 액션 워크플로우가 위치하는 곳입니다. 

.github/workflows

여기에 워크플로우(workflow) 파일을 YAML로 작성해서 넣으면 되며 파일 이름은 딱히 상관 없으며 확장자 .workflow 를 붙여주면 됩니다.

깃헙 사이트에서도 생성할 수 있으며 이 경우 GUI의 도움을 받을 수 있기 때문에 그다지 어렵지 않게 생성할 수 있습니다.

워크플로우

name: [WORKFLOW_NAME]

on: [EVENT - 이 워크플로우가 어떤 이벤트에 동작하는가]

jobs:
  [JOB_NAME]:
    runs-on: [RUNNER - 어떤 OS에서 동작하는가]
    env:
      [ENV KEY]: [VALUE]
      [ENV KEY]: [VALUE]
      ...
    steps:
    - name: [ACTION_NAME]
      uses: [ACTION]
      ...

EVENT

대표적으로 저장소에 코드가 푸시뒬 경우가 있습니다. 

on: push

깃헙 자체의 액션 생성 기능에서 풀다운으로 고를 수도 있으므로 굳이 다 알 필요는 없을 것 같습니다.

RUNNER

아마도 우분투 리눅스를 가장 많이 쓸 것 같습니다. 

runs-on: ubuntu-latest

ACTION

액션의 경우는 워낙 다양해서 깃헙 마켓 플레이스에서 검색하시는 것을 추천합니다. 관련된 매개변수도 액션에 따라 바뀌기도 해서 여기서 정리하는 것은 매우 힘든 일입니다.

물론 이 외에 커스텀으로 가장 많이 사용하게 될 쉘 커맨드의 경우 아래와 같은 식으로 사용됩니다. 

- name: run some shell commands
  working-directory: some-directory
  run: |
    echo "multi"
    echo "line"
    echo "shell"
    echo "commands"

working-directory 는 필요할 때만 명시하면 됩니다.

관련글

  • 🌏 깃헙 액션으로 경계가 명확한 CI/CD 파이프라인 구현하기
  • 🌏 GitHub Action과 Poetry를 사용한 파이썬 패키지 개발

Footnotes:

1

Continuous Integration, 자동화 프로세스의 일종이지만, 저장소에 코드가 올라오면 자동으로 에러를 체크하는 등의 보조 수단으로도 자주 활용하는 것 같습니다.
