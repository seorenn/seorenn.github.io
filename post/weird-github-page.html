왜 GitHub Pages의 일부 글이 나타나지 않았을까?

2020년 4월 14일 작성

seorenn-publish를 개발하며 테스트 하던 도중의 사건이다. 내 프로젝트에는 post와 note라는 두 개의 하위 디렉터리가 있다. 이 둘은 당연히 다른 용도의 글을 올리기 위한 것이다. 일단 출력 된 HTML 파일들을 내 개인 GitHub Pages 저장소에 올리면 곧 파일을 웹 브라우저로 볼 수 있었다.

문제의 등장

별 생각 없이 지내다가 이상한 증상을 만났다. note 링크를 누르면 최근 수정한 순서대로 글 목록이 보여야 한다. 그런데 이상한 test 라는 문구만 표시되는 증상이 나타났다.

뭔가 이상해서 일부러 이상한 수정을 한 뒤 다시 커밋해서 푸시하는 작업을 수 차례 수 시간에 걸쳐 반복해봤다. 그래도 여전히 test가 나타났다.

이상해서 note 아래에 있는 글 주소를 입력해보니 이번에는 404가 나타났다. 심지어 note 아래의 모든 글이 이렇게 404로 없다는 표시가 떴다.

하지만 내 깃헙 저장소에 파일은 정상적으로 잘 올라가 있는 것을 확인했다. 깃헙 측에서도 디플로이 했다는 메시지가 남아있기도 했다.

삽질의 시작

note의 내용이 전부 보이지 않는 것은 심각한 증상이다. 하지만 이상한 점은 일부 디렉터리의 내용만 제대로 보이지 않는다는 점이었다. 특히 저 test 라는 문구가 나타나는 이유를 전혀 몰랐었다.

구글링을 해봐도 별 다른 힌트를 찾을 수 없었다. 전부 GitHub Status를 보거나 혹은 무료 플랜으로 개인 저장소에 올린 거 아니냐는 지극히 당연한 듯한 답변만 찾을 수 있었다.

시험 삼아 아예 note 디렉터리를 지워봤다. 그래도 test가 나타났다. 지우고 다른 내용으로도 채워봤다. 그래도 test가 나타났다.

처음에는 GitHub의 정적 사이트 제너레이터가 내가 기억에도 없는 테스트 용으로 올린 파일의 내용을 아직도 캐싱하고 보여 주는 것일까 하는 생각만 들었다. 그런데 이런 저런 시험을 해 보다 보니 이쯤 되면 뭔가 유령이 있는 거 아닐까 하는 생각이 들 정도였다.

해결

그러다 구글링 도중에 발견했던 단어 하나가 떠올랐다.

gh-pages

gh-pages 라는 브랜치는 각 저장소에서 GitHub 페이지에 홈페이지를 만들 수 있게 지원하는 특수한 브랜치다. 여기다 웹 페이지 파일을 올리기만 하면 user.github.io/projectname 이런 주소로 접속할 수 있는 웹 페이지를 서비스해 준다.

아 잠깐. 혹시 나 note 라는 이상한 프로젝트에 gh-pages 라는 브랜치로 뭔가 올려 뒀었나?

…

정말로 있었다.

언제 만들었는지 기억도 안 나지만 note라는 이름의 저장소가 있었고 여기의 gh-pages 라는 브랜치에 test라고 적어놓은 index.html 파일이 올려져 있었다.

…

발견하고는 미치도록 환장한다는 느낌 보다는 이상하게 기쁜 느낌이 들었다. 아 분명 나는 이 note라는 프로젝트 저장소를 만든 기억이 없는데 '그래도 찾았다는 게 어딘가' 하는 희열이 다행히도 짜증을 앞섰다.

어쨌든 사뿐하고 과감하게 해당 note라는 이름의 저장소를 날려버렸다. 그리고 웹 브라우저에서 새로고침을 눌러봤다.

잘 나온다. 문제가 해결되었다. 하루라는 시간이 날아가 버렸지만 뭐 어떠하리.

결론

GitHub Pages의 특정 디렉터리가 이상하게 보인다면 혹시 자신이 올린 다른 프로젝트 저장소의 gh-pages 브랜치가 있는지 점검해보자.
